## 피연산자
- 산술 명령어의 피연산자에는 제약이 있다.
    = 레지스터에 직접 구현된 것만 사용가능

### 레지스터
in MIPS, 32 bit
- 개수가 한정되어 있다.
- 현대 컴퓨터에는 MIPS처럼 보통 32개의 레지스터가 있음
    = 기호 형태로 표현된 MIPS 언어를 단계적으로 구체화 할 때, 산술 명령어의 각 피연산자는 `32`개의 32비트 레지스터 중 하나여야 한다.
- `작은것이 더 빠르다`
    = 레지스터 개수를 제한하는 이유
    = 레지스터가 아주 많아지면 전기 신호가 더 멀리까지 전달되어야 한다.
    = 클럭 사이클시간이 길어진다.
    = 결국 trade off
    (더 많은 레지스터를 원하는 프로그램 vs 클럭 사이클)
- 레지스터를 효율적으로 사용하는 것은 프로그램 성능에 매우 중요하다.
- $xx (MIPS convention for register)

### 메모리 피연산자
- 값 하나만 기억하는 `단순 변수`외에도 복잡한 자료구조들이 있다. ex. 배열, 구조체
    = 레지스터 개수보다 훨씬 많은 데이터 원소가 있을 수 있다.
    especially, memory can have numerous data as many as billions, whereas register only like few
    = 배열이나 구조체 같은 자료구조는 `메모리`에 보관한다.

- 산술연산은 레지스터에서만 실행되므로, `메모리 <-> 레지스터`간 데이터를 주고받는 명령어가 있어야 한다.
    = `데이터 전송 명령어`
- 메모리에 기억된 `데이터 워드`에 접근하려면 `메모리 주소`를 지정해야 한다.

**메모리**
- 주소가 인덱스 역할을 하는 큰 1차원 배열
- 메모리 주소는, 명령어의 상수부분 + 두 번째 레지스터의 합

### 적재
- 메모리 -> 레지스터로 데이터를 `복사`해오는 데이터 전송명령
- `연산자 이름` + `메모리에서 읽어온 값을 저장할 레지스터` + `메모리 접근에 사용할 상수와 레지스터`
```
g = h + A[8];

g in $s1, h in $s2
A's address in $s3

lw  $t0, 8($s3) // $t0: temp register
add $s1, $s2, $t0
```
- 데이터 전송 명령어의 상수부분 ex. 8 above -> `offset`
- 주소 계산을 위해 여기에 더해지는 레지스터 ex. $s3 -> `base register`

### 저장
- 레지스터에서 메모리로 데이터를 보내는 것
    = `레지스터 -> 메모리`
- 저장명령의 생김새는 적재와 같다.
    = 연산자 이름, 저장할 데이터를 갖고 있는 레지스터 + 배열 원소 선택에 사용할 변위 + 베이스 레지스터
- 주소의 일부는 `상수`형태로 명령어에 포함 + 일부는 `레지스터`에 기억
- `sw`(store word)

NOTE)
변수를 레지스터와 연관짓는 일 외에도, 배열 or 구조체 같은 자료구조를 `메모리`에 할당하는 것도 컴파일러의 임무
- 메모리에 할당 후, 자료구조의 `시작주소`를 `데이터 전송명령`에 넣는다.
(데이터 전송명령: 레지스터 <-> 메모리 간 데이터를 주고받는 명령어)
- 대부분 바이트 단위로 주소지정
(워드주소는 워드를 구성하는 4바이트 주소중 하나를 사용한다.)

### big-end, and little-end
- 제일 왼쪽, `최상위 바이트 주소`를 워드주소로 사용하는가
    vs 제일 오른쪽, `최하위 바이트 주소`를 워드주소로 사용하는가

**spilling register**
- 자주 사용하지 않는 변수를 메모리에 넣는 일
(컴파일러는 자주 사용되는 변수를 가능한 한 많이 레지스터에 넣고, 나머지는 메모리에 저장했다가 필요할 때 꺼내서 레지스터로 가져온다.)
- 좋은 성능을 위해서는 컴파일러가 레지스터를 효율적으로 활용해야 한다.
    = 작을수록 빠르다. (메모리 > 레지스터 in size, < in speed)
    = 레지스터에 저장된 데이터는 사용하기도 간편하다.
    (데이터 연산을 바로할 수 있기 때문에 == 데이터 전송명령은 읽기/쓰기만 할뿐, 연산은 불가능)
    = 레지스터는 메모리보다 접근시간이 짧고, 처리량도 많다.
    (레지스터에 있는 데이터 사용 == 시간 절약)
    = energywise, 레지스터 접근 < 메모리 접근 in terms of consumption

NOTE)
상수는 프로그램이 적재될 때, `메모리`에 넣어진다.

**수치(immediate) 피연산자**
- 피연산자 중 하나가 상수인 산술 연산 명령어를 제공하는 것
    = 적재 명령을 사용하지 않는다.
- 상수 피연산자는 자주 사용되므로, 매번 메모리에서 가져오는 데 따르는 비용, 에너지 소모 등을 줄여줌

**상수 0의 역할**
- 명령어에 여러 가지 유용한 변형을 제공
    -> 명령어 집합을 단순하게 해준다.
ex. mov: 피연산자 중 하나가 0인 add ($zero)

### 명령어의 컴퓨터 내부표현
- 컴퓨터 내부에서는 결국 높고 낮은 `전기 신호의 연속`
    = 숫자로 표현할 수 있다.
    = 실제로 명령어의 각 부분을 숫자로 볼 수 있음
    = 그리고 이 숫자들을 나란히 늘어놓으면 명령어가 된다.
- 거의 모든 명령어가 레지스터를 사용하기 때문에, 레지스터 이름을 `숫자`로 매핑하는 규칙이 있어야 한다.

### MIPS 명령어의 필드
op: 6bits
- 명령어가 실행할 연산의 종류, 연산자

rs: 5bits
- 첫 번째 근원지 피연산자 레지스터

rt: 5bits
- 두 번째 근원지 피연산자 레지스터

rd: 5 bits
- 목적지 레지스터. 연산 결과가 기억된다.

shamt: 5 bits
- 자리이동 량 (shift)

funct: 6 bits
- 기능
- op 필드에서 연산의 종류를 표시
- 이후 funct 필드에는 그중의 한 연산을 구체적으로 지정한다.
    = 기능코드

5 ~ 6 bit로는 부족할 수 있다.
ex. `lw`: 레지스터 필드 2개 + 상수 필드 하나 필요
- trade off between 모든 명령어의 길이를 같게 and 명령어 형식 통일
- `적당한 절충`이 필요한 순간
    = 모든 명령어의 길이를 같게 하되, 명령어 종류에 따라 `형식`은 다르게
- R타입: 5~6비트
- I타입: 16비트
    = 수치 연산과 데이터 전송 명령어에 사용

ex.
op + rs + rt + 나머지 16비트
1) r타입 3개로 채우거나
2) i타입 하나로 채우거나

- 명령어 형식은 `첫 번째 필드의 값`을 보고 구분할 수 있다.
    = 형식별로 op 필드가 가질 수 있는 값이 다르다.
    = 요 필드를 보고 오른쪽 절반이 R타입인지 I타입인지 구분한다.


two base essential theories
1. 명령어는 숫자로 표현된다.
    = 프로그램이 이진수 파일 형태로
2. 프로그램은 메모리에 기억되어 있어서, 데이터처럼 읽고 쓸 수 있다.
     = 내장 프로그램
     = 메모리에 여러 프로그램이 기억되고 이것을 언제든지 가져와서 쓸 수 있다.

### 논리 연산 명령어
1. shift
- 비트를 왼쪽/오른쪽으로 이동
- 이동후 빈자리는 0/1(?)로 채워준다.
(책에는 0이라고 되어있지만, 음수의 경우 >> 연산 시 오른쪽 비트에 1을 채워줬던 것 같기도,,,)
```
sll $t2, $s0, 4 // reg $t2 = reg $s0 << 4 bits
```
R형식의 shamt 필드
= shift amount

2. AND
- 두 비트가 1일 경우면 결과가 1
- bit mask: 0의 위치에 해당하는 비트들을 강제로 0으로 만들어줄 수 있다.

3. OR
- 두 비트 중 하나라도 1이면 1

4. NOT
- 비트 뒤집기
    = 0 -> 1, 1 -> 0

### 판단을 위한 명령어
- if, goto, ...

ex. beq register1, regster2, L1
- regester 1, 2의 값이 같으면 L1로!
- beq: branch if equal
- bne: branch if not equal

**comparison between signed and unsigned numbers**
- 부호 있는 정수를 부호 없는 정수처럼 다루면 0 <= x < y 검사 비용을 낮출 수 있다.
    = 인덱스가 배열의 한계를 벗어났는지 확인하는 검사에 딱 맞는다.
    = 핵심은 2의 보수로 표현된 음수가 부호없는 정수로 보면 매우 큰 수 처럼 보인다는 것

### 순환문
with while, for

(조건절 대소 비교 시)
- slt: set on less than
```
slt $t0, $s3, $s4 == t0 = 1 if $s3 < $s4
```

- slti: slt + 상수 피연산자
```
slti $t0, $s2, 10 == t0 =  if s2 < 10
```
    = sltu: set on less than unsigned
    = sltiu: set on less than immediate unsigned
