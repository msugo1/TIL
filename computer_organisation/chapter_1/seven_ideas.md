### 1. abstraction (추상화)
: 설계의 단순화
- 여러 수준에서 설계를 명시
- 하위 수준의 상세한 사항을 감춰, 상위 모델을 단순화

### 2. common case fast
- 자주 생기는 일을 빠르게 만드는 것 > 드물게 생기는 일 최적화 in performance improvement

### 3. parallelism (병렬성)
- 병렬적으로 작업을 수행하여 성능을 높이도록 설계

### 4. pipelining (파이프라인)
- 각 단계들을 파이프처럼 연결

### 5. prediction (예측)
- 예측은 복구비용이 상대적으로 저렴
- 예측이 맞을 확률이 높다면, 일단 시작하는 것이 무조건 좋은 경우가 만다.

### 6. 메모리 계층구조
in 최상위
- 가장 빠르고, 가장 작고, 비트당 가격이 제일 비싼 메모리 사용

in 최하위
- 느리지만 크고 비트당 가격이 제일 싼 메모리

### 7. dependability(신용도)
: 여유분을 이용한 신용도 개선
(like spare tires - 하나가 고장나도 여유분을 통해 정상운용)
