### 컴퓨터 하드웨어는 아주 단순한 저수준의 명령만 실행이 가능하다.
- starts from complex 응용
- all the way down to simple instructions
- 높은 수준의 작업을 단순한 컴퓨터 명령어로 번역 혹은 해독하는 여러 겹의 소프트웨어가 필요하다.
    = 계층적 구성

핵심은 **운영체제** 와 **컴파일러**
### 1. OS
- 사용자 프로그램과 하드웨어 간의 인터페이스 역할
- 각종 서비스와 감독 기능제공
ex.
```
* 기본적 입출력 작업의 처리
* 보조기억장치 및 메모리 할당
* 컴퓨터를 동시에 사용하는 여러 응용들 간의 컴퓨터 공유방법 제공
```

### 2. 컴파일러
- 상위 수준의 언어로 작성된 프로그램을 `하드웨어가 실행할 수 있는 명령어`로 번역
    = 프로그래밍 언어는 복잡
    = but 기계가 이해할 수 있는 가장 쉬운 신호는 on/off, which is represented as 1/0
    = 이진수
    = bit

**어셈블러**
- 기호식 표현 to 이진수 번역 프로그램
- 컴퓨터가 수행할 명령어를 한 줄에 하나씩 쓴다.
    = 프로그래머에게 기계식 사고방식 강요
- 추후에 고성능 언어 작성 -> 컴퓨터 명령어로 변역 개념의 등장(컴파일)
    = 고수준 -> 컴파일 -> 어셈블

## to Hardware
### 모든 하드웨어의 기본기능
- 데이터 입력/출력/처리/저장

### 컴퓨터의 고전적 구성요소 다섯가지
1. 입력
2. 출력
3. 메모리
4. 데이터 패스 -------> 프로세서
5. 제어유닛 ----------

### 프로세서
- 프로그램의 지시대로 일을 하는 부분
- CPU: Central Processor Unit
- 데이터 패스 + 제어유닛으로 구성

**데이터 패스**
- 연산 수행

**제어유닛**
- 명령에 따라 데이터 패스, 메모리, 입출력 장치가 할일 지시


### 메모리
**DRAM**
- Dynamic Random Access Memory
- 랜덤 접근 = 메모리의 어떤 부분을 읽든지 같은 시간이 걸린다.

**캐시 메모리**
- DRAM의 버퍼 역할
- 작고 빠른 메모리
- SRAM(= static random access memory)

**SRAM - DRAM**은 메모리 `계층구조`를 형성한다.

### ABI
= application binary interface
- 응용 프로그래머에게 제공되는 기본 명령어 집합 + 운영체제 인터페이스

### 컴퓨터 구조와 구현의 분리
- `명령어 집합구조` 덕에 가능하다.
ex. 디지털 시계의 기능 vs 디지털 시계 하드웨어

---
### 계층구조
- 하드웨어, 소프트웨어 모두 `추상화`를 이용하여 계층적 구성
- 각 하위계층의 세세한 부분이 상위 계층에서는 보이지 않도록 한다.
- 가장 중요한 것은 하드웨어와 하위 계층 소프트웨어 간의 인터페이스인 `명령어 집합구조`
- 인터페이스 추상화를 통해 같은 소프트웨어를 가지고 여러 구현을 만들어낼 수 있다.

## 컴퓨터 간의 통신
- 컴퓨터 네트워크를 통해서 컴퓨터 전체가 서로 연결, 사용자의 계산능력 확장!

### 네트워크로 컴퓨터를 연결했을 때 장점?
1. 통신
- 컴퓨터끼리 고속으로 정보를 주고받을 수 있다.

2. 자원공유
- 컴퓨터마다 입출력 장치를 따로 갖출 필요가 없다.
- 네트워크 상의 모든 컴퓨터가 입출력 장치를 공유할 수 있다.

3. 원격접근
- 컴퓨터들을 연결하면 멀리 떨어져 있는 컴퓨터도 쉽게 이용할 수 있다.

### 단점
- 거리가 멀 수록 성능저하
