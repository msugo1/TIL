### What OSs do
- 컴퓨터에서 `항상` 실행되는 프로그램
    = `커널`
    = 외에도 `미들웨어` 및 `시스템 프로그램` 포함
- control and manage hardwares around multiple users
    = 효율적이고 공정하게 시스템을 운영할 수 있도록 어느 요청에 자원을 할당할지 결정
    = 컴퓨터 부적절한 사용을 방지하기 위해, 사용자 프로그램의 수행 제어
- 다른 프로그램이 유용한 작업을 할 수 있는 `환경`제공

**시스템 프로그램**
- 운영체제와 관련 but 반드시 커널의 일부일 필요는 없음
- 시스템 실행 중, 시스템을 관리하는데 도움

**응용 프로그램**
- 시스템 작동과 관련되지 않은 모든 프로그램

## Computer-System Organization
### 장치 컨트롤러
- 일부 로컬버퍼 저장소 및 특수 목적 레지스터 집합을 유지, 관리
- 제어하는 주변 장치 <-> 로컬 버퍼 저장소 (데이터 이동)

### 장치 드라이버
- 각 장치 컨트롤러마다 있다.
- 장치 컨트롤러의 작동을 잘 알고 있으며, 나머지 운영체제에 장치에 대한 일관된 인터페이스를 제공한다.
- CPU와 장치 컨트롤러는 병렬로 실행
    = 메모리 사이클을 놓고 경쟁
    = 메모리 컨트롤러: 메모리에 대한 액세스 동기화

### 인터럽트
입출력 시,
1. 장치 드라이버: 장치 컨트롤러의 적절한 레지스터에 값 적재
2. 장치 컨트롤러: 레지스터의 내용 검사 후, 수행할 작업 결정 (ex. 키보드 문자 읽기)
3. 컨트롤러: 장치에서 로컬 버퍼로 데이터 전송 시작
4. 데이터 전송 완료 시, 장치 컨트롤러 -> 장치 드라이버 `완료`신호 보냄

now
(읽기 요청) 데이터 또는 데이터에 대한 포인터 반환 by 장치 드라이브 and 운영체제의 다른 부분에 제어 넘김

(다른 요청) 장치 드라이버: `쓰기 완료` or `장치 사용중`과 같은 상태정보 반환

**작업을 완료한 경우 `인터럽트`를 통해 알린다. (컨트롤러 -> 장치 드라이버)**
- 시스템 버스: 주요 구성요소 사이의 주요 통신 경로
    = 하드웨어는 어느 순간이든 시스템 버스를 통해, CPU에 신호를 보내 인터럽트 발생시킬 수 있음
- CPU 인터럽트 시, CPU는 하던 일 중단 -> 고정된 위치로 실행 옮김
    = 고정된 위치: 인터럽트를 위한 서비스 루틴이 위치한 시작주소 (mainly)
- 인터럽트 서비스 루틴 실행
    = 인터럽트는 적절한 서비스 루틴으로 제어 전달 (call a handler)
- 완료 후, 인터럽트 된 연산 재개
- 인터럽트는 매우 빈번하게 발생하므로 빠르게 처리되어야 한다.
    = 필요한 속도를 제공하기 위해, 인터럽트 루틴에 대한 포인터들의 `테이블`을 대신 이용할 수 있음
    (중간 루틴 둘 필요X. 테이블을 통하여 간접적으로 인터럽트 루틴 호출)

```
raise(other devices) -> catch(CPU) -> dispatch to interrupt handler(CPU) -> clear
```

- 인터럽트 번호 = 인덱스
- 필요한 상태 저장 before dispatched
- after clear, `return_from_interrupt`
    = CPU를 인터럽트 전 실행 상태로 되돌린다.

### 더욱 정교한 인터럽트 처리
1. 중요한 처리 중 인터럽트 처리 연기
2. 더욱 효율적인 방법으로 핸들러 디스패치
3. 다단계 인터럽트 필요
    = 우선순위 별로 인터럽트 구분
    = 적절한 긴급도로 대응하기 위해서

### nonmaskable vs maskable interrupt
1. 마스크 불가능 인터럽트
- 복구할 수 없는 메모리 오류와 같은 이벤트를 위해 예약된 인터럽트

2. 마스크 가능 인터럽트
- 장치 컨트롤러's 서비스 요청

### interrupt chain
- 벡터 방식 인터럽트 기법?
    = 핸들러 검색 오버헤드 줄이기
- 실제로 컴퓨터에는 인터럽트 벡터 주소의 개수보다 더 많은 장치 존재
    = 요 문제를 해결하기 위한 것이 인터럽트 체인
    = 인덱스 헤드로 가서 링크되어진 핸들러 따라가면서 호출(적절한 핸들러 발견할 때까지)

### 워드
- 컴퓨터 구조 본연의 데이터 단위
ex. 64비트 레지스터 and 메모리 주소 지정 - 64비트 워드

### DMA(Direct Memory Access)
- 직접 메모리 액세스
    = NVS I/O와 같은 대량 데이터 이동에 사용시 발생하는 오버헤드를 낮추기 위함
    (인터럽트 구동 I/O의 형태는 소량의 데이터를 이동하는데 좋다. but 대량의 경우 오버헤드!)
- 장치에 대한 `버퍼 및 포인터, 입출력 카운트` 세팅
    -> 장치 제어기는 `CPU의 개입 없이` 
        메모리로부터 자신의 버퍼장치 or 버퍼로부터 메모리로 `데이터 블록 전체 전송`

### NUMA(non-uniform memory access)
- 로컬 메모리 액세스 할 때 뿐만 아니라, 시스템 상호 연결에 대한 경합이 없음
    = 더 많은 프로세스가 추가될수록 효과적으로 확장가능
- 시스템 상호연결을 통해 원격 메모리에 액세스해야할 때 지연 시간이 증가 -> 성능저하
    = but, 운영체제의 CPU 스케줄링 및 메모리 관리를 통해 단점을 최소화할 수 있다.
- 많은 수의 프로세서를 수용할 수 있도록 확장 가능
    = 고성능 컴퓨팅 시스템 and 서버에서 점점 인기

### 블레이드 서버
- 다수의 처리기 보드 및 입출력 보드, 네트워킹 보드들이 하나의 섀시 안에 장착되는 형태
- 각 블레이드 처리기 보드는 독립적으로 부팅될 수 있고, 자기 자신의 운영체제를 수행
- 근본적으로 여러 독립적인 다중 처리기 시스템으로 구성

## 운영체제의 작동
- 컴퓨터의 전원을 켜거나 재부팅 할 때와 같이 컴퓨터를 실행하려면, `초기 프로그램`이 실행되어야 한다.
    = 일반적으로 컴퓨터 하드웨어 내에 펌웨어로 저장
    = CPU 레지스터, 장치 컨트롤러, 메모리 내용에 이르기까지 시스템의 모든 측면을 초기화
- 부트 스트랩 프로그램은 운영체제를 적재하는 방법, 해당 시스템 실행을 시작하는 방법을 알아야 한다.
    = 운영체제 커널을 찾아 메모리에 적재
- 커널이 적재되어 실행되면, 시스템과 사용자에게 서비스 제공가능
    = 일부 서비스는 커널이 실행되는 전체시간 동안 실행되는 `시스템 데몬`이 되어야 한다.
    = 부팅할 때 메모리에 적재되는 `시스템 프로그램`에 의해 `커널 외부`에서 제공된다.
    = `systemd`: 첫 번째 시스템 프로그램 in Linux (and executes other many daemons)

### 프로세스 vs 프로그램
- 프로그램: 디스크에 저장된 파일. 수동적
- 프로세스: 다음에 수행할 명령을 지정하는 PC(Programme Counter)를 지정하는 능동적인 개체

### 입출력 서브시스템
- UNIX 입출력 장치의 특성은 입출력 서브시스템에 의해 운영체제 자제의 대부분으로 숨겨져있다.
    = 일반적인 장치 드라이버 인터페이스 + 특정 하드웨어 장치들을 위한 드라이버

## 가상화
- 단일 컴퓨터(CPU, 메모리, 디스크 드라이브, 네트워크 인터페이스 카드 등)의 하드웨어를 여러 가지 실행 환경으로 추상화
    -> 개별 환경이 자신만의 컴퓨터에서 실행되고 있다는 환상을 만드는 기술
- 다양한 운영체제 간에 전환 가능

**에뮬레이션**
- 소프트웨어로 컴퓨터 하드웨어 시뮬레이션
    = 소스 CPU 유형이 대상 CPU 유형과 다른 경우에 사용
    = 비용이 크다.
    (ex. 소스와 목표 CPU가 비슷한 성능 수준일 경우, 에뮬레이트된 코드는 원래의 코드보다 훨씬 느리게 수행 된다.)

- 가상화에서는 특정 CPU를 위해 컴파일된 운영체제가 동일 CPU용의 다른 운영체제 내에서 수행된다.

### VMM(가상머신 관리자)
- 게스트 운영체제 수행, 그들의 자원이용 관리
- 각 게스트들을 서로로부터 보호

### 비트맵
- n개의 항의 상태를 나타내는데 사용가능한 n개의 이진 비트 스트링
    = 공간효율성!
    = 대량의 자원의 가용성을 표시할 때 일반적으로 사용
    (ex. 001011101 - i 번째 비트는 i 번째 자원과 연관. 0은 가용 1은 불용 ...)
