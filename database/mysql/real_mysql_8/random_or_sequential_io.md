## SSD vs HDD
1. SSD
- 플래시 메모리
    = 디스크 원판 회전X for search

2. HDD
- 원판(플래터)

**랜덤 I/O** 발생 시 둘의 성능 차이가 넘사벽!
- SSD는 탐색 시간이 줄어들기 때문에 성능이 HDD보단 좋지만, 여전히 순차 I/O에 비해 성능저하
- RDBMS는 일반적으로 랜덤 I/O가 많이 발생
- 랜덤 I/O를 줄여서 `꼭 필요한` 데이터만 읽도록 하는 것이 쿼리 튜닝

**디스크에 데이터를 읽고 쓰는데 걸리는 시간은**
- 디스크 헤더를 움직여서, 읽고 쓸 위치로 옮기는 단계에서 결정

## 랜덤 IO vs 순차 IO
- 디스크 헤더를 1번만 움직이냐 vs 여러번 움직이냐
- 성능차이 발생지점
- 그룹커밋, 로그버퍼 등이 존재하는 이유
- SSD도 랜덤I/O, 순차I/O 차이가 존재한다.

NOTE) 랜덤 I/O, 순차 I/O 모두 파일 쓰기 실행 시 동기화 필요
- 동기화 작업이 빈번하면 순차I/O 또한 성능저하
- 이 부분을 해결하는데 쓰이는 것이 `RAID`컨트롤러, especially 캐시메모
